# Executive Communication Frameworks (Phase 3 only)

Use these **only** once the user has explicitly asked to synthesise, structure,
or write something up, and alignment (Phase 2) is confirmed. The goal is a
persuasive, decision-oriented artefact — not a summary of the conversation.

Apply them in this order: **Governing Thought → SCQA/SCR → Minto Pyramid → MECE check.**

## 1. Governing Thought

The single sentence the audience must remember. One idea, stated as an
actionable position, not a topic.

- ❌ Topic: "Our pricing of audits."
- ✅ Governing Thought: "Replace free audits with a bounded, paid Discovery sprint."

If you cannot state it in one sentence, the thinking is not ready — return to Phase 2.

## 2. SCQA (long form) / SCR (short form)

Frames *why the audience should care* before the answer.

| Element | SCQA | What it does |
|---------|------|--------------|
| **S**ituation | Shared, uncontested context | Establishes common ground |
| **C**omplication | What changed / the tension | Creates the need to act |
| **Q**uestion | The question the complication raises | Names the decision |
| **A**nswer | The Governing Thought | Resolves it |

**SCR** (Situation → Complication → Resolution) is the compressed form for
emails, slide titles, and verbal openers — drop the explicit Question.

**Template:**
> *Situation:* We win deals partly on free ontology audits.
> *Complication:* They anchor price low and expose our method — our scarcest asset.
> *Question:* How do we keep conversion without giving away strategy?
> *Answer (Governing Thought):* Replace free audits with a bounded, paid Discovery sprint.

## 3. Minto Pyramid

Structure beneath the Governing Thought: top-down, answer first.

```
            Governing Thought (the answer)
           /            |              \
   Supporting       Supporting       Supporting
    line 1           line 2           line 3
   /     \          /     \          /     \
 evidence evidence ...    ...      evidence evidence
```

Rules:
- Lead with the answer; reasons follow (not the reverse).
- Each level **summarises** the level below it.
- Group ideas so each group answers a single question raised by the level above.

## 4. MECE check (Mutually Exclusive, Collectively Exhaustive)

Test the supporting lines:
- **Mutually Exclusive:** no overlap between lines (no double-counting).
- **Collectively Exhaustive:** together they cover the whole question (no gaps).

If two lines overlap, merge them. If something important fits nowhere, the
grouping is wrong — re-cut it.

## 5. Make implications explicit

A consulting artefact is not complete until it states:
- **Trade-offs** chosen and rejected.
- **Risks** and how they are bounded.
- **Implications** — what changes for strategy, services, and tactics, kept clearly separate.

## Format selection

| Audience / artefact | Recommended frame |
|---------------------|-------------------|
| Board / partners decision | Governing Thought + SCQA + 3-line Minto |
| Client proposal | SCQA + Minto, trade-offs and stopping points explicit |
| Email / Slack / opener | SCR + Governing Thought only |
| One-page canvas | Governing Thought as title; MECE lines as sections |
